# b1CodingTool: Global Context
This is the root `agent.md` file managed by b1CodingTool. 
It contains project-agnostic software development practices and general notes.

## General Guidelines
- Write clean, modular code.
- Include thoughtful documentation.

## Commands

```bash
uv run b1 --help                     # Show CLI help
uv run b1 init                       # Bootstrap a project with .agent/ structure
uv run b1 install <module-source>    # Install a module (local path or git URL)
uv run b1 pull                       # Sync modules from upstream repo
uv run b1 push                       # Draft PR to contribute generalized rules upstream
uv run b1 pair                       # Regenerate CLAUDE.md / GEMINI.md / CODEX.md
uv run b1 dashboard                  # Launch React dashboard on localhost
```

## Architecture

```
src/b1/
  cli.py               # Typer app entry point — registers all commands
  commands/            # One file per CLI command (init, install, pull, push, pair, dashboard)
  core/
    compiler.py        # Compiles agent.md hierarchy into a single string
    translator.py      # Writes compiled string to CLAUDE.md, GEMINI.md, CODEX.md
    context_manager.py # Scaffolds agent.md files on b1 init
    installer.py       # Copies module files into .agent/modules/, runs skill setup scripts
    fetcher.py         # Resolves module source: local path or git clone to ~/.b1/cache/
    scaffolder.py      # Scaffolds .agent/ directory structure
    schema.py          # Pydantic models: ModuleConfig, SkillConfig
    config.py          # Reads/writes .agent/config.yaml (B1Config model)
  server/main.py       # FastAPI server that serves data to the React dashboard
dashboard/             # React frontend (Vite + TypeScript)
docs/                  # Planning docs and SPEC.md
```

## Gotchas
- CLAUDE.md, GEMINI.md, and CODEX.md are AUTO-GENERATED by `b1 pair` — edit `agent.md` or `.agent/project/agent.md` instead, then re-run `b1 pair`.
- Modules must contain `b1-module.yaml` (or `module.yaml`) at their root to be recognized.
- Remote module cache lives at `~/.b1/cache/` (git clones); re-install pulls latest.
- Config state stored at `.agent/config.yaml` — tracks `upstream_repo` and `active_agents`.
- `uv` is used for all Python dependency management and running — not pip or poetry.
